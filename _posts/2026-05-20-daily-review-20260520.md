## SungWoo824

**LiveCMD 역할별 사용 권한 적용 (OR-1415)** — 백엔드와 프론트를 한 번에 재설계한 큰 작업.

- 백엔드: `FeaturePermissionStore`에 `allowed_roles` 필드를 도입하고, 서버 기동 시 `ensure_default()`로 전체 역할을 자동 채움. 로그인 응답에 `feature_permission`을 포함하고, 역할 삭제 시 cascade 정리. 신규 `/v3/acl/feature/live-cmd/check` UseCase 추가(super_admin/is_root 면제).
- 프론트: 403 처리를 `CustomEvent` 방식에서 **외부 스토어 + `useSyncExternalStore` 패턴**으로 전환. 다중 엔드포인트 누적 + 중복 제거를 지원하고, 권한 오류 처리 흐름을 no-throw로 재설계.

## smahn9123

**FastAPI v3 파일 I/O aiofiles 전환 (OR-1422)** — 동기 파일 I/O를 비동기로 바꿔 이벤트 루프 블로킹을 제거. `os.path.*`/`pathlib.Path` 8건을 `aiofiles.os.*`로 교체하고, 호출처를 모두 `await`로 동기화. redundant stat 제거(EAFP 전환)까지.

**Playwright MCP 도입 (OR-1361)** — manager-web UI 자동 검증 환경. chromium/headless/legacy 3채널 등록 + 290줄 가이드 문서.

그리고 **전날 코드 리뷰 피드백을 정확히 반영(OR-1414)** — mosquitto-preload 셰임의 ABI 호환성(glibc ceiling)과 미로드 시 fail-safe 동작을 README에 명시.
