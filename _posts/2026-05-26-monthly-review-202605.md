## smahn9123 — 서버·DevOps 전방위

**mosquitto SSL_OP_IGNORE_UNEXPECTED_EOF preload shim 패키징 동봉 (OR-1414)**.
OpenSSL 3.0.x record layer 버그(partial TLS record + close 에러 시 SSL_CTX 공유 상태 손상)를 Ubuntu 24.04가 패치하지 않는 점을 격리 환경에서 확정한 뒤, `SSL_OP_IGNORE_UNEXPECTED_EOF` 플래그를 `LD_PRELOAD`로 주입하는 shim을 운영에 안착시켰다. 5월에는 폐쇄망 update 안정성을 위한 3단계 보강을 마무리했다 — ① silent skip 방지 안전망(install.sh 실패 시 setup/update가 die 대신 경고만), ② WSL Ubuntu 24.04(glibc 2.39)에서 빌드한 사전 빌드 `.so`를 git에 동봉 + Makefile `MAX_GLIBC=2.39` ceiling을 `make verify`가 강제, ③ Orange-Package CLAUDE.md에 트리·동봉 정책·setup/uninstall 단계 문서화. 운영 서버 `.so`와 byte-identical(sha256) 확정.

**fail2ban 패키지 통합 — firewall.rules `__SSH_PORT__` 토큰화 (OR-1313)**.
인터넷 노출 서버 SSH 무차별 대입을 systemd journal 백엔드 + nftables banaction + 지수 증가 ban + recidive 메타-jail(24h 내 5번 ban → 1주 장기차단)로 차단. install.sh / setup.sh / update.sh / uninstall.sh에 일관 통합하고, firewall.rules에 `__SSH_PORT__` 토큰을 도입해 install 시점에 `detect_ssh_port` 헬퍼가 동적 치환하도록 만들어 고객사마다 다른 sshd 포트를 자동 반영. 폐쇄망 update에서 fail2ban 미설치 환경도 graceful skip.

**Service 계층의 API 스키마 의존 제거 — 순환 import 근본 정리 (OR-1425)**.
Service/Infra가 API 스키마(`app/api/v3/schemas/*`)를 거꾸로 import하던 layer 위반 3건(CommandTemplate·Organization·infra/cgroup)을 모두 정상화. 각 계층 전용 `schemas/` 디렉터리에 입력/결과 모델 신설, `usecases/mapper/` 3개(CommandTemplate·Organization·Health)로 API↔내부 변환을 일관화. `__future__` + `TYPE_CHECKING` 회피책을 완전히 걷어내고도 외부 API 응답 변경 0건 — 순수 내부 리팩터링.

**nodedetect 엔드포인트 v3 마이그레이션 — distinctCount + totalCount 두 의미 분리 (OR-1424)**.
`/v2/nodedetect/count(group_by_cuid_cgid=true)`를 `/v3/detect/nodes/count`로 옮기며 응답을 '고유 증상 인스턴스 수(distinctCount, 기존 v2 의미)'와 '기간 내 발생 횟수(totalCount, 1시간 버킷 누적 합산)' 둘로 명확히 분리. 1회 aggregate로 두 값 동시 산출, `get_date_range_obj` DI로 tz 누락 시 422 자동 검증. manager-web 호출부 마이그레이션 + dead 위젯·dead code 정리 + '기간 내' 접두사 라벨 통일까지 한 묶음.

## wychoi-orangelabs — 커널 드라이버·MS 서명 필터링

**커널 드라이버 SE_SIGNING_LEVEL 한계 분석 + PE Authenticode 추출 방향 (OR-1423)**.
Medium IL의 MS 서명 user-mode 앱(Code.exe, Teams.exe, Edge 메인, Office)은 OS가 CI 강제를 생략해 `SignatureLevel`이 영원히 0임을 Windows 11 26100.8493 실측(86건 SL=0 vs 0건 SL≥1)으로 확정. `PsSetLoadImageNotifyRoutineEx` + `PS_IMAGE_NOTIFY_CONFLICTING_ARCHITECTURE` 교체는 옳지만 실효가 제한적임을 인정하고, ProcessNotify 시점에 IL Gate(Medium/High이고 SL<8일 때만) → PE 임베디드 서명 PKCS#7 파싱 → CN prefix 'Microsoft' 매칭 → `PROCESS_ENTRY` 캐시하는 Stage 2 구조를 설계. 파일 이벤트당 비용 0, ProcessNotify 1회 PE 헤더 ~수 KB I/O로 한정.

**WebView2 가상 호스트 매핑 — file:// → https://orange.local (OR-1404)**.
`orange.user.exe`의 WebView2가 `file://` 스킴을 opaque origin으로 취급해 같은 문서 안의 fragment navigation조차 cross-origin으로 분류되며 보안 경고가 source 라벨 두 개로 이중 출력되던 문제를, WebView2 가상 호스트 매핑(`https://orange.local`)으로 정식 origin (https, orange.local, 443) 튜플 위에서 풀었다. 프론트엔드 우회(`createMemoryHistory`, `about:blank`, v-if 가드)로는 스킴 정책을 제어할 수 없어 스킴 자체를 옮긴 정식 해결책. opaque origin이 막던 기능을 정상 origin 위에서 풀어주는 작업.

**Live Agent 파일 삭제·오류 모니터링 — 3개 탭 동일 패턴 정렬 (OR-1406)**.
신규/오류/삭제 3개 탭이 동일한 MQTT 명령 흐름(`get.{COMMAND}_SELECT_PERIOD` 초기 요청 → T1 실시간 갱신, `alert:false`로 이탈 시 Agent 구독 해제)으로 동작하도록 정렬. 공유 그리드 컴포넌트가 `gridId`로 컬럼 분기(오류 탭만 `ErrorDesc` flex), `getRowId`는 `SUID-FilePath-FileName` 조합. 8초 타임아웃 폴백으로 데이터 없을 때 무한 스피너 방지, `suppressCellFocus`로 행 클릭 UX 정리, `rowData={[]}` 리터럴 prop이 매 렌더마다 `applyTransaction({add})` 추가 행을 초기화하던 버그픽스까지.

**파일 삭제·오류 필터링 시간 기반 throttle + LRU table 설계 (OR-1427)**.
기존 `count % 100 == 1` throttle은 한 key가 지속 폭주해도 100건당 1건씩 무한 통과되어 폭주 자체를 못 막던 한계를, `KeQueryUnbiasedInterruptTime` 기반 시간 단위(10분당 1회)로 전환. hash table + `ERESOURCE`를 LRU array(512 entry, `EX_PUSH_LOCK` 8B)로 교체해 포화 시 전체 reset이 throttle을 무력화하던 문제까지 해소. 17조 회 케이스 기준 1,700억 건 notify → 4,320건으로 약 4천만배 절감. 시간 source가 sleep을 제외해 절전 직후 폭주도 차단.

**Live Agent 증상 클릭 무한 로딩·SPROCESS_PROCESS 미발송 (OR-1347, OR-1363, OR-1365)**.
SUID/DUID만 있는 행을 클릭했을 때 빈 프로세스 탭이 무한 로딩으로 빠지던 회귀를 추적해 SPROCESS_PROCESS 발송 누락과 컬럼별 차등 동작 통합까지 한 묶음으로 정리. 사용자 일관성 관점에서 행 클릭 시 컬럼별 동작을 재설계.
